💡 Why 12-Bit Dodecet?

🎯 16x More States

4,096 distinct values vs 256 for 8-bit. This means 16x better precision for geometric operations, color representation, and discrete calculations.

Example: Color gradients with 16x smoother transitions

🔢 Hex-Friendly

Each dodecet = exactly 3 hex digits. Perfect for hex editors, debugging, and human-readable representation. No padding, no awkward conversions.

Example: 0xABC instead of 0xAB, 0xCD

📐 Geometric Optimization

12 bits divide evenly into 3 nibbles, perfect for 3D coordinates (x, y, z). Each axis gets 4 bits of precision in a single dodecet.

Example: 3D point (x,y,z) in one value

⚡ Calculus-Friendly

Natural step sizes for numerical derivatives and integrals. Fine-grained representation enables better numerical analysis.

Example: Derivatives at 1/4096 resolution

💾 Storage Efficient

341 values per bit vs 32 for 8-bit. 8.3x more efficient per bit. Store more data in less space without losing quality.

Example: 16x compression for coordinates

🚀 Performance

SIMD-friendly operations. Bitwise operations on 12-bit values are faster than floating-point for many geometric calculations.

Example: < 1ns for dodecet creation

🔧 Technical Details

Dodecet Structure

┌─────────────────────────────────────────┐
│           DODECET (12 bits)             │
├─────────────────────────────────────────┤
│  Nibble 2  │  Nibble 1  │  Nibble 0    │
│  (4 bits)  │  (4 bits)  │  (4 bits)    │
│  [11:8]    │  [7:4]     │  [3:0]       │
├─────────────────────────────────────────┤
│  Example:   0xA        0xB        0xC  │
│  Hex: 0xABC = 1010 1011 1100 (binary)  │
│  Decimal: 2748                             │
└─────────────────────────────────────────┘

Representation Formats

Hex: ABC (3 digits)
Binary: 1010 1011 1100 (12 bits)
Decimal: 2748 (0-4095)
Normalized: 0.6709 (0.0-1.0)
Signed: 700 (-2048 to 2047)
