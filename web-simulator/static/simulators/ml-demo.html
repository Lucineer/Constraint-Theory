Working Code Examples

// Dodecet encoding for ML weights
import { Dodecet } from './dodecet';

// Encode a weight value as 12-bit dodecet
const weight = 0.732;
const encoded = new Dodecet(Math.round((weight + 1) * 2048) & 0xFFF);

console.log(encoded.toHexString());  // "CDB" (12-bit hex)
console.log(encoded.toSigned() / 2048);  // 0.732 (decoded)

// Batch encoding for weight matrices
const weights = [0.5, -0.3, 0.8, -0.1];
const encodedWeights = weights.map(w =>
  new Dodecet(Math.round((w + 1) * 2048) & 0xFFF)
);

// Feature map point encoding
interface FeaturePoint {
  x: Dodecet;  // 12-bit X coordinate
  y: Dodecet;  // 12-bit Y coordinate
  label: string;
}

const point: FeaturePoint = {
  x: new Dodecet(0x800),  // center X
  y: new Dodecet(0xC00),  // upper Y
  label: 'sample_1'
};

// Calculate cosine similarity between encoded points
function cosineSimilarity(p1: FeaturePoint, p2: FeaturePoint): number {
  const x1 = p1.x.normalize(), y1 = p1.y.normalize();
  const x2 = p2.x.normalize(), y2 = p2.y.normalize();

  const dot = x1 * x2 + y1 * y2;
  const mag1 = Math.sqrt(x1 * x1 + y1 * y1);
  const mag2 = Math.sqrt(x2 * x2 + y2 * y2);

  return dot / (mag1 * mag2);
}

Important Limitations

What This Demonstrates

These visualizations show how dodecet encoding can be used for ML-related tasks. They demonstrate the encoding precision and visualization capabilities of the Constraint Theory framework.

This is NOT:

  • A replacement for standard ML frameworks (TensorFlow, PyTorch)
  • Proven to improve ML model performance
  • Benchmarked against production quantization methods (PQ, OPQ)
  • Suitable for production ML workloads without extensive validation

Current limitations:

  • 2D-only (higher dimensions require extension)
  • No GPU acceleration for large-scale operations
  • Not validated on real ML datasets
  • Comparison data is illustrative, not benchmarked
